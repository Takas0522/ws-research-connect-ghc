# TeamsやSharePointの情報を集約してドキュメントを作ってもらうプロンプト例

``` markdown
TeamsでやりとりされているSaaS管理アプリについて
Teams上のやり取りやSharePoint上のドキュメントをもとに、SaaS管理アプリのドキュメントを作成してください。
SharePoint上のドキュメントはPowerPointとExcelの資料となっています。Skillを使用して中身の解析を行い必要な情報を抽出してください。
PowerPoint上のドキュメントは図表が多く構成されているため、SVGやPNGに変換するなどして情報を参照できるようにします。
ドキュメントはユーザーの利用や提供する価値について記載するビジネス仕様とシステムの仕様について記載するシステム仕様の2点を含めてください。
それぞれ長いファイルになることを避ける為セクションごとにファイルを分割して作成してください。
ドキュメントは `docs/specs/{開発名}` の下に保存してください。
```
※いい感じのTeamsの会話はtools/配下のアプリで作成できるが、EntraIDアプリの権限が付与できないなどの問題がある場合は `workshops-docs/resources` のディレクトリが参考にできる（Teamsのやり取りのRawデータがそのまま格納されている）
