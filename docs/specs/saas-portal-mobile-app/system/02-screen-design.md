# 画面構成・画面遷移

## 画面遷移図（PoC スコープ）

```
スプラッシュ
    ↓
ログイン / サインアップ
    ↓ （認証成功）
ホーム（ダッシュボード）
    ├─→ サービス利用状況詳細
    │       └─→ 月次推移グラフ
    ├─→ アプリ一覧
    │       └─→ アプリ起動（Mock）
    └─→ 設定 / プロフィール
            └─→ ログアウト
```

## 画面一覧

### 1. ログイン / サインアップ画面

- メールアドレス・パスワードによるログイン
- 新規ユーザー登録（テナントコード入力）
- パスワードリセット
- JWTトークン取得・保存

### 2. ホーム（ダッシュボード）画面

テナントに紐づくSaaSサービスの利用状況を一覧表示する。

#### 表示要素

- **テナント情報:** テナント名、契約サービス数
- **利用状況サマリーカード:** サービスごとの主要メトリクスをカード形式で表示
  - サービス名
  - 当月利用量 / 無料枠
  - 利用率（プログレスバー）
  - 前月比（増減アイコン）
- **利用者数推移グラフ:** 直近12ヶ月の月次推移（折れ線グラフ）
- **利用目的別サマリー:** 利用目的（primary_use_case）ごとの集計（円グラフ）

#### デザイン方針（サトウ氏の要件）

> 「数字の羅列だと現場の方は見てくれないので、グラフと表の両方でパッと直感的に把握できるUIがかなり重要」

- グラフとテーブルを組み合わせた直感的なUI
- スマートフォンの画面サイズに最適化されたレイアウト
- スワイプ操作でサービス間を切り替え可能

### 3. サービス利用状況詳細画面

特定サービスの詳細な利用状況を表示する。

#### 表示要素

- サービス基本情報（サービス名、プラン名、契約期間）
- 月次利用推移グラフ（折れ線 / 棒グラフ切替）
- 利用量テーブル（月次データ一覧）
- 課金情報（当月/前月の請求額比較）

### 4. アプリ一覧 / 起動画面

テナントが契約しているSaaSアプリの一覧を表示し、ワンタップで起動する。

#### 表示要素

- アプリアイコン + サービス名
- 契約ステータス（利用中 / 利用停止）
- 起動ボタン（PoCではMock実装）

#### PoC実装方針（大川氏）

> 「PoC段階ではアプリ起動のところはMockで実装して、ディープリンク連携は次フェーズに回す」

- PoCではボタンタップ時にMock画面を表示
- Phase 1でディープリンクまたはカスタムURLスキームによる実アプリ起動を実装

### 5. 設定 / プロフィール画面

- ユーザープロフィール表示・編集
- テナント情報表示
- 通知設定（Phase 1）
- ログアウト
