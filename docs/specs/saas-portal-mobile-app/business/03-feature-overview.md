# 機能概要

## 機能一覧

本アプリの機能は大きく **利用状況ダッシュボード**、**アプリ起動ポータル**、**認証・ユーザー管理** の3領域に分かれる。

### 利用状況ダッシュボード

- テナントごとの月次利用状況一覧表示
- 利用者数の推移グラフ（折れ線・棒グラフ）
- 利用目的別の傾向分析（円グラフ・サマリー表）
- サービス別利用量の比較表示
- 直感的なUI（「数字の羅列ではなくグラフと表の両方で把握」）

### アプリ起動ポータル

- 契約中のSaaSアプリ一覧表示
- ワンタップでの各アプリ起動
- サービスステータス表示（利用中 / 利用停止 / 解約済）

### 認証・ユーザー管理

- テナント単位のユーザー管理
- JWTベースのサインアップ・ログインフロー
- テナント管理者 / 一般ユーザーのロール分離

## フェーズ別リリース範囲

### PoC（技術検証）— Android版先行

| 機能 | 概要 | 優先度 |
|---|---|---|
| JWT認証基盤 | テナント/ユーザー認証、トークンリフレッシュ | **高** |
| テナント別ダッシュボード | 利用者数推移グラフ + 利用状況表 | **高** |
| Mockアプリ起動 | 各SaaSアプリへの起動ボタン（Mock実装） | **高** |

**方針（PMワタナベ氏）：**
> 「PoCのスコープは①JWT認証基盤、②テナント別ダッシュボード（グラフ+表）、③Mockアプリ起動の3つに絞ろう。まずAndroid版で検証してからiOSに展開する段階的アプローチ」

### Phase 1 — iOS版追加・リアルアプリ連携

| 機能 | 概要 | 優先度 |
|---|---|---|
| iOS版リリース | Swift + SwiftUI での実装 | **高** |
| ディープリンク連携 | 実アプリへのディープリンク起動 | **高** |
| プッシュ通知 | 利用量アラート・更新通知 | 中 |

**開発者（大川氏）：**
> 「PoC段階ではアプリ起動のところはMockで実装して、ディープリンク連携は次フェーズに回す方針」

### Phase 2 — 機能拡張（Feature Flag）

| 機能 | 概要 | 優先度 |
|---|---|---|
| Feature Flag管理 | 各SaaSサービスのベータ機能ON/OFF切り替え | 中 |
| ベータ機能利用 | フラグONでベータ機能を利用可能に | 中 |
| フィードバック収集 | 開発途中でのユーザーフィードバック取得 | 低 |

**Feature Flag の目的：**
- 開発完了後の大規模修正を避け、開発途中でフィードバックを得ることで効率的な開発を実現
- 機能実装の価値を早い段階で確認し、開発の本格化判断を初期段階で行う
- フラグは機能ごとに管理され、管理者がON/OFFを切り替え可能
