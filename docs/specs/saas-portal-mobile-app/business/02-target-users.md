# 想定ユーザーと提供価値

## 想定ユーザー

### プライマリユーザー：テナント管理者

自社のConnectシリーズSaaSサービスを契約している顧客企業の管理者・IT担当者。

- テナント内のサービス利用状況を把握する責務を持つ
- 利用量の推移を確認し、プラン変更や追加契約の判断材料を得たい
- 複数のSaaSサービスを一元的に管理したい

### セカンダリユーザー：テナント一般ユーザー

テナントに所属する一般利用者。

- 自身が所属するテナントの利用状況をグラフで直感的に確認したい
- 利用している各SaaSアプリにワンタップでアクセスしたい

## 提供価値

| ユーザー | 提供価値 |
|---|---|
| テナント管理者 | テナント全体の利用状況を横断的に把握でき、プラン最適化やコスト管理の判断材料を得られる |
| テナント一般ユーザー | 自社が契約しているSaaSサービスの利用傾向を直感的に確認でき、各サービスに素早くアクセスできる |
| 自社営業チーム（間接） | 顧客自身がデータを把握することで、提案時の共通認識形成が容易になり営業効率が向上する |

## ユーザーの声（Teams議論より）

> 「お客さんから『契約しているSaaSの利用状況をスマホでサッと確認したい』という声をよくいただいている」 — サトウ スズキ（営業担当）

> 「テナントごとの利用者数の推移や、どんな目的で使われているかの傾向分析。数字の羅列だと現場の方は見てくれないので、グラフと表の両方でパッと直感的に把握できるUIがかなり重要」 — サトウ スズキ（営業担当）
