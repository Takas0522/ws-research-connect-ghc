# 開発フェーズ・マイルストーン

## フェーズ概要

全体を4つのフェーズに分けて段階的にリリースする。

```
Phase 1: 基盤構築 + マスタ管理
    ↓
Phase 2: 契約管理 + 利用実績
    ↓
Phase 3: ダッシュボード + 分析
    ↓
Phase 4: トライアル機能
```

## Phase 1: 基盤構築 + マスタ管理

### 目標

データベーススキーマの構築と、製品・顧客の基本的な CRUD を実装する。

### タスク

| # | タスク | レイヤー | 依存 |
|---|--------|---------|------|
| 1-1 | DB スキーマ作成（全テーブル） | Database | - |
| 1-2 | シードデータ SQL 作成 | Database | 1-1 |
| 1-3 | EF Core モデル定義 | Backend | 1-1 |
| 1-4 | 製品 API（CRUD） | Backend | 1-3 |
| 1-5 | 課金プラン API（CRUD） | Backend | 1-3 |
| 1-6 | 顧客 API（CRUD） | Backend | 1-3 |
| 1-7 | React Router 導入 + レイアウト | Frontend | - |
| 1-8 | 製品一覧・詳細・登録画面 | Frontend | 1-4, 1-7 |
| 1-9 | 顧客一覧・詳細・登録画面 | Frontend | 1-6, 1-7 |
| 1-10 | 製品 API ユニットテスト | Test | 1-4 |
| 1-11 | 顧客 API ユニットテスト | Test | 1-6 |

### 成果物

- 全テーブルのスキーマ + シードデータ
- 製品・プラン・顧客の CRUD API
- 製品管理画面・顧客管理画面

| 2-4 | 利用実績 API（登録 + 一括登録 + 一覧） | Backend | 2-3 |
| 2-5 | 契約一覧・詳細画面 | Frontend | 2-1 |
| 2-6 | プラン変更・解約 UI | Frontend | 2-1, 2-2 |
| 2-7 | 利用実績登録画面（請求額プレビュー付き） | Frontend | 2-4 |
| 2-8 | 利用実績一覧画面 | Frontend | 2-4 |
| 2-9 | BillingService ユニットテスト | Test | 2-3 |
| 2-10 | 契約 API 統合テスト | Test | 2-1 |
| 2-11 | 利用実績 API 統合テスト | Test | 2-4 |

### 成果物

## 依存関係図

```
Phase 1 ──→ Phase 2 ──→ Phase 3
                │              │
                └──→ Phase 4 ──┘
                     (Phase 2 以降で着手可能)
```

## リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 課金ロジックの複雑性 | 高 | Phase 2 の BillingService テストを最優先で作成 |
| データ移行（Excel → DB） | 中 | シードデータ SQL で既存データを正確に再現 |
| グラフライブラリの学習コスト | 低 | recharts は軽量で習得容易 |
| 要件の追加・変更 | 中 | フェーズ分割で変更の影響範囲を限定 |
