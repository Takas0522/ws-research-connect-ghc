# 画面構成・機能要件

## 画面構成案

![画面構成案](../images/sekkei_image2.png)

大川氏が提案した4画面構成をベースとし、ワタナベPMの「まずダッシュボードと従量データ取込からプロトを作る」方針に合わせて、プロトタイプは 1 と 4 を優先開発する。

## 共通要件

| 項目 | 内容 |
|---|---|
| ナビゲーション | 上部グローバルナビで4主要画面を切り替える |
| ローディング | API取得中はスケルトンまたはローディング表示を行う |
| Empty状態 | データ未登録時は次の操作（登録 / 取込）を案内する |
| エラー表示 | API失敗、CSV検証エラー、権限不足は画面上で明示する |
| 最終更新日時 | ダッシュボードと一覧画面では最終更新日時を表示する |
| アクセス制御 | `sales` は担当顧客のみ更新可、`admin` は全件閲覧 / 更新可 |

## 1. ダッシュボード画面

### 概要

営業が日常的に確認するメイン画面。顧客ごとの利用率を一覧表示し、超過リスクをアラートで通知する。

### 画面要素

| 要素 | 内容 |
|---|---|
| 利用状況サマリ | 顧客別利用率の一覧表示（カードまたはテーブル形式） |
| 超過アラート一覧 | プラン閾値超過時の警告バッジ。既定値は90% |
| 月次トレンドグラフ | 直近12ヶ月の利用推移を Recharts で表示 |
| 利用目的サマリ | `primary_use_case` ごとの件数 / 利用量を集計 |
| 最終更新日時 | 月次CSV反映日時を表示 |

### ユースケース

1. 営業担当者がログイン後、ダッシュボードで担当顧客の利用状況を確認
2. 超過アラートが出ている顧客を確認し、プラン見直し提案を検討
3. 月次トレンドと利用目的サマリを見て、次回提案の切り口を整理

### 補足要件

- グラフ対象メトリクスは Phase 1 では `api_calls` を優先し、必要に応じて他メトリクスを切り替え表示する
- 月次更新であることが分かるよう、リアルタイム監視を想起させる表現は避ける
- データ欠損月は 0 または未取込として表示し、グラフが途切れないようにする

## 2. マスタ管理画面

### 概要

管理者が製品・プラン・顧客・メトリクス・ユーザーを登録 / 管理する画面。一覧 → 詳細 → 編集のフローを基本とする。

### 管理対象タブ

| タブ | 内容 | 主な項目 | 権限 |
|---|---|---|---|
| 製品 | SaaS製品の基本情報 | 製品コード、製品名、カテゴリ、ベンダー | admin |
| プラン | プラン別料金 / 閾値 / 上限値 | プランコード、月額基本料、閾値、従量上限 | admin |
| 顧客 | 契約主体の情報 | 顧客コード、顧客名、担当営業、連絡先 | admin |
| メトリクス | 従量課金指標の定義 | metric_code、表示名、単位 | admin |
| ユーザー | 社内利用者管理 | email、表示名、role、利用可否 | admin |
| 監査ログ | 変更履歴参照 | 実行者、操作種別、対象、日時 | admin |

> メトリクス定義を独立させることで、製品ごとに異なる従量項目を後から追加しやすくする。

## 3. 契約管理画面

### 概要

顧客ごとの契約内容・ステータスを管理する画面。営業担当者は担当顧客のみ更新できる。

### 機能

| 機能 | 内容 |
|---|---|
| 契約一覧 | 契約状況を一覧表示（顧客名 / 製品 / プラン / 更新日で検索可能） |
| 契約詳細 | 契約開始日・終了日・更新日・月額基本料・ライセンス数・利用目的の表示 / 編集 |
| 更新時期アラート | 契約更新日が近い顧客を通知 |
| プラン変更履歴 | ライセンス数・プラン・単価変更履歴を時系列で表示 |
| 契約ステータス管理 | `active` `renewing` `suspended` `terminated` を管理 |

### 補足要件

- 契約変更時は `contract_plan_history` を必ず残す
- Phase 1 では月中単価変更の**自動按分は行わず**、履歴記録と月次確定時の確認に留める
- 営業担当者が担当外顧客の編集画面に直接アクセスした場合は権限エラーを返す

## 4. 従量データ取込画面

### 概要

月次の従量課金データをCSVで取り込む画面。Phase 1ではCSV手動アップロード、Phase 2でAPI自動連携を想定。

### 取込ステップ

| ステップ | 内容 |
|---|---|
| 1. ファイル選択 | ドラッグ&ドロップまたはファイル選択でCSVをアップロード |
| 2. バリデーション | マスタ突合、対象月、重複、数値範囲を検証 |
| 3. プレビュー | 取込件数、差分、エラー件数を確認 |
| 4. 確定 | 問題なければ保存し、ダッシュボードへ反映 |

### 機能

| 機能 | 内容 |
|---|---|
| CSVアップロード | 月次従量データの取り込み |
| 取込結果プレビュー | 対象月、件数、差分、エラー内容の事前確認 |
| バリデーション | マスタとの突合、重複チェック、数値範囲チェック |
| 再取込（置換） | 同一 `billing_month` のデータは確認後に置換 |
| 取込履歴参照 | 過去の成功 / 失敗履歴と実行者を確認 |

### 運用フロー

- 更新は毎月1〜5営業日（JST）を原則とする
- 締め日超過後の再取込は管理者のみ許可できる
- エラー時は保存せず、担当営業がCSVを修正して再アップロードする
- 取込済みデータの再実行は `usage_imports` に履歴を残し、差分が追えること
