# 試用（トライアル）機能仕様

## 概要

顧客が本契約前に SaaS 製品を無料で試用できるトライアル機能。
Teams での議論（2026-03-14）に基づき、顧客の意思決定スピードを加速し、契約促進を図る。

## ビジネス価値

### 顧客側のメリット

- 無料で製品を試した上で契約判断ができる（投資判断のリスク低減）
- 複数製品の比較検討が気軽にできる

### 自社側のメリット

- 契約判断の加速 → 契約周期の短縮
- 試用体験による信頼獲得 → 解約リスクの低減
- 実装コストに対して大きなリターン（テナント管理画面の改修が主）

## トライアルの種類

### 制限レベル

| レベル | 説明 | 例 |
|--------|------|-----|
| 制限なし（full） | 本契約と同等の全機能を利用可能 | Enterprise プラン相当の体験 |
| 機能制限（feature_limited） | 一部機能を制限した状態 | エクスポート不可、API連携不可 等 |
| 容量制限（capacity_limited） | 利用量に上限を設けた状態 | 5ユーザーまで、100GB まで 等 |

### 製品別トライアル設定（推奨）

| 製品 | 推奨トライアル期間 | 推奨制限レベル | 備考 |
|------|------------------|--------------|------|
| CloudSync Pro | 14日 | 容量制限（5ユーザー） | Starter 相当 |
| DataVault | 14日 | 容量制限（100GB） | Basic 相当 |
| BizFlow | 30日 | 容量制限（100件/月） | Standard 相当 |
| InsightDash | 14日 | 機能制限（3ダッシュボード） | Team 相当 |
| SecureGate | 30日 | 容量制限（10ユーザー） | セキュリティ製品のため慎重に |
| ChatAssist AI | 14日 | 容量制限（500会話） | ベータ版のため広めに |
| FormBuilder | - | トライアル不要 | Free プランが実質トライアル |

> **注**: 上記は推奨値。実際の設定はトライアル作成時に個別指定できるようにする。

## トライアルのライフサイクル

```
[作成]
  │
  ▼
進行中 (active)
  │
  ├──→ 転換済み (converted)  ※ 本契約に移行
  │
  ├──→ 期限切れ (expired)    ※ 終了日を過ぎた（自動）
  │
  └──→ キャンセル (cancelled) ※ 手動で中止
```

### ステータス遷移ルール

| 現在のステータス | 遷移先 | トリガー |
|----------------|--------|---------|
| 進行中 | 転換済み | UC-21: 管理者が本契約に転換 |
| 進行中 | 期限切れ | UC-22: 終了日を過ぎた時点で自動 |
| 進行中 | キャンセル | 管理者が手動でキャンセル |
| 転換済み | - | 最終状態（変更不可） |
| 期限切れ | - | 最終状態（変更不可） |
| キャンセル | - | 最終状態（変更不可） |

## 課金ルール

### TR-01: トライアル中は課金なし

- トライアル期間中の請求額は **0円** とする
- 利用実績（利用量）は通常通り記録する

### TR-02: 本契約転換時

- 転換日をもって本契約の開始日とする
- トライアル中の利用実績は本契約の請求には含めない
- トライアルの `converted_contract_id` に新規契約の ID を設定する

### TR-03: 期限切れ時

- 製品の利用を停止する（アプリ側での制御は v1 スコープ外）
- トライアルステータスを「期限切れ」に変更する
- 再度のトライアル作成は可能とする（同一顧客・同一製品でも期限切れ後なら可）

## UI 上の表示

### トライアル一覧画面

- 進行中のトライアルを優先表示
- 残日数を色分け表示:
  - 7日以上: 通常表示
  - 3〜6日: 黄色（警告）
  - 1〜2日: 赤（緊急）
  - 0日以下: グレーアウト（期限切れ）

### ダッシュボードへの影響

- トライアル中の顧客は売上集計に含めない（0円のため）
- ダッシュボードに「トライアル → 本契約 転換率」の KPI を表示する
- 転換見込みのあるトライアル（残日数少 × 利用量多い）をハイライト表示する

## 実装フェーズ（Teams 議論より）

| フェーズ | 内容 | 見積期間 |
|---------|------|---------|
| Phase 1 | テナント管理画面改修（トライアルの CRUD） | 1週間 |
| Phase 2 | 売上・原価ロジック分岐（トライアル時は0円） | 1週間 |
| Phase 3 | ダッシュボード対応（トライアル KPI 表示） | 0.5〜1週間 |

> Phase 1〜3 合計: 2〜3週間
