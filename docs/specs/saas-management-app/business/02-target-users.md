# 想定ユーザーと提供価値

## 想定ユーザー

本アプリは**社内用アプリケーション**として開発する。主な利用者は以下の2ロール。

### 営業担当者

- **日常業務:** 顧客別の利用状況確認、契約内容の参照・更新、月次データの取り込み
- **現在の課題:** 担当顧客のExcelを個別管理しており、月末集計に丸1日以上かかる
- **利用範囲:** 担当顧客の契約・利用実績の参照と更新、月次CSV取込、ダッシュボード確認

### 管理者

- **日常業務:** 全顧客の横断的な利用状況モニタリング、マスタデータ管理、ユーザー管理
- **現在の課題:** 最新版ファイルの特定が困難、全社集計が属人化
- **利用範囲:** 全機能へのアクセス、ユーザー管理、締め日超過時の取込再開、監査ログ確認

## 提供価値

### 営業担当者にとっての価値

| 価値 | 詳細 |
|---|---|
| **一元管理** | 担当者ごとのExcel管理から脱却し、全データをWebアプリに集約 |
| **データ可視化** | 月次更新後の利用推移グラフと利用目的別の集計で営業提案の説得力を向上 |
| **超過アラート** | 契約プランに設定した閾値を超えた顧客をダッシュボードで把握 |
| **プラン見直し提案** | 従量実績データに基づくプラン見直し候補を抽出 |

### 管理者にとっての価値

| 価値 | 詳細 |
|---|---|
| **集計自動化** | CSV取り込みで月次集計を標準化し、手動転記を排除 |
| **引き継ぎ容易化** | 担当替え時も顧客・契約・履歴を同一画面で追跡可能 |
| **運用統制** | 取込履歴、監査ログ、ユーザー管理で属人化を抑止 |
| **契約管理** | 更新時期のアラートで契約更新漏れを防止 |

## Phase別の提供価値

| フェーズ | 提供価値 |
|---|---|
| プロトタイプ | ダッシュボードとCSV取込の最低限の業務適合性を確認する |
| MVP | 4主要画面と管理者機能により、月次運用をExcelからWebへ移行する |
| Phase 2 | SaaS API連携、自動按分、分析高度化により運用負荷をさらに下げる |

## ビジネスインパクト

Teams上の議論（サトウ氏の発言）によると、直近3ヶ月でABC商事やGHIテクノロジーズなど複数社でAPI呼出が上限に近づいている状況があり、プラン見直し提案のタイミングとなっている。本アプリにより：

- 超過リスクのある顧客を**月次更新後すぐに検知**し、営業が次回提案に反映可能
- 利用推移のグラフ化により、**データドリブンな提案**を実現
- 月末集計の**工数削減**（丸1日 → 取込と確認を含めて数十分程度を目標）
- 担当変更時も履歴と運用ログが残るため、**引き継ぎコストを削減**
