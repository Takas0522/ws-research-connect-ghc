# 開発スケジュール

## スケジュール概要

![開発スケジュール案](../images/image7.png)

全体で約5ヶ月のスケジュールを想定。2026年4月開始、8月リリースを目標とする。

## フェーズ一覧

| フェーズ | 期間 | 工期 |
|---|---|---|
| 要件定義・設計 | 4月〜5月中旬 | 1.5ヶ月 |
| バックエンド開発 | 4月下旬〜6月 | 2ヶ月 |
| フロントエンド開発 | 5月〜7月 | 2ヶ月 |
| テスト・修正 | 7月 | 1ヶ月 |
| リリース | 8月 | GO |

## フェーズ詳細

### Phase 1: 要件定義・設計（1.5ヶ月）

- 詳細要件定義（現行Excel運用のヒアリング含む）
- マスタデータ設計（製品・プラン・顧客・メトリクス定義）
- API設計（FastAPI エンドポイント設計・自動ドキュメント生成）
- 画面ワイヤーフレーム

### Phase 1 → Phase 2: プロトタイプ

Teams議論で決定された方針として、まずダッシュボード（①）と従量データ取込（④）からプロトタイプを作成する。

### Phase 2: バックエンド開発（2ヶ月）

- FastAPI + MongoDB によるAPI実装
- JWT認証基盤
- CSVインポート機能
- データバリデーション

### Phase 3: フロントエンド開発（2ヶ月）

- React + TypeScript によるSPA構築
- ダッシュボード（Rechartsによるグラフ）
- マスタ管理画面CRUD
- 契約管理画面

### Phase 4: テスト・修正（1ヶ月）

- 結合テスト・UAT
- 営業部門によるユーザーテスト
- フィードバック反映

### リリース

- 8月中の本番リリースを目標
- 来週のレビュー会でv2企画書を最終確認し、GOかどうか判断（ワタナベPMの方針）

## 将来構想（Phase 2以降）

Phase 2として、SaaS製品APIからの自動データ取得を見据えている。

- CSVの手動取り込みを廃止し、月次バッチまたはWebhook連携でデータを自動取得
- 大川氏の提案：「まずCSV取込でプロトを作り、Phase 2でAPI自動連携も見据える」
