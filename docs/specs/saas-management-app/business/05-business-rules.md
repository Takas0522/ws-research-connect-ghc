# ビジネスルール・課金ロジック

## 課金モデルの分類

現行の SaaS 製品には以下の3つの課金モデルが存在する。

### 固定料金型（Flat-Rate）

利用量に関わらず月額基本料のみで課金する。

- **対象製品**: CloudSync Pro, InsightDash, FormBuilder
- **計算式**: `請求額 = 月額基本料`

### 従量課金型（Usage-Based）

月額基本料に加え、無料枠を超過した利用量に対して従量課金する。

- **対象製品**: DataVault, BizFlow, ChatAssist AI
- **計算式**: `請求額 = 月額基本料 + MAX(0, 利用量 - 無料枠) × 従量単価`

### ハイブリッド型（Hybrid）

月額基本料に加えユーザー数に応じた従量課金。

- **対象製品**: SecureGate
- **計算式**: `請求額 = 月額基本料 + MAX(0, ユーザー数 - 無料枠) × 従量単価`

## 請求額計算の詳細ルール

### BR-01: 基本計算フロー

```
1. 契約のプラン情報を取得する
2. 従量単価が null の場合 → 固定料金型として月額基本料を返す
3. 従量単価が設定されている場合:
   a. 利用量から無料枠を引く（負の場合は0）
   b. 超過分 × 従量単価を計算
   c. 月額基本料 + 従量課金額 = 請求額
4. 請求額は円単位（小数切り捨て）
```

### BR-02: 年契割引

- 契約形態が「年契」かつプランに年契割引率が設定されている場合、月額基本料に割引を適用する
- **計算式**: `割引後月額基本料 = 月額基本料 × (1 - 割引率 / 100)`
- 割引は月額基本料にのみ適用し、従量課金部分には適用しない

### BR-03: トライアル中の課金

- トライアル中の顧客には課金しない（請求額 = 0）
- トライアル中の利用量は記録するが、請求額の自動計算では 0 円とする
- 詳細は [06-trial-feature.md](06-trial-feature.md) を参照

## 契約に関するビジネスルール

### BR-04: 契約の一意性

- 同一顧客・同一製品で同時にアクティブな契約は1つだけ
- プラン変更は既存契約の更新として処理する（新規契約は作成しない）

### BR-05: プラン変更の記録

- プラン変更時は必ず ContractHistory に記録する
- 記録項目: 変更前プラン、変更後プラン、変更理由、変更日時
- プラン変更は即時反映とする（月途中でも変更可能）

### BR-06: 解約処理

- 解約時は契約ステータスを「解約済み」に変更し、end_date を設定する
- 解約理由を ContractHistory に記録する
- 解約後のデータ（過去の利用実績・請求履歴）は保持する（論理削除）

### BR-07: 月次利用実績の登録

- 同一契約・同一月の利用実績は 1 レコード
- 再登録した場合は上書き更新する（UPSERT）
- 利用量の登録と同時に請求額を自動計算する

## 計算例

### 例1: BizFlow Standard（月契、利用 162件）

```
月額基本料:  ¥10,000
無料枠:     100件
超過分:     162 - 100 = 62件
従量課金:   62 × ¥50 = ¥3,100
請求額:     ¥10,000 + ¥3,100 = ¥13,100
```

### 例2: SecureGate Advanced（年契、295ユーザー）

```
月額基本料:  ¥60,000（年契割引なし）
無料枠:     200ユーザー
超過分:     295 - 200 = 95ユーザー
従量課金:   95 × ¥80 = ¥7,600
請求額:     ¥60,000 + ¥7,600 = ¥67,600
```

### 例3: CloudSync Pro Business（年契、42ユーザー）

```
月額基本料:  ¥15,000
年契割引:   10% → 割引後 ¥13,500
無料枠:     50ユーザー（枠内）
従量課金:   ¥0
請求額:     ¥13,500
※ 注: 現行データでは ¥15,000 と記載されており、年契割引が適用されていない可能性あり
        → 要確認事項として確認する
```
