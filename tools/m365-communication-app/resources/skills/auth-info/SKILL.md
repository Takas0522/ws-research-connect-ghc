# 認証情報取得スキル

## 目的

指定されたペルソナの認証情報（アクセストークン）を取得します。

## 動作ルール

1. `get_auth_token` ツールを使用して、指定されたペルソナのキャッシュされたアクセストークンを取得してください
2. トークンが期限切れの場合、ツール内部で自動的にリフレッシュトークンを使用した更新が試みられます
3. リフレッシュも失敗した場合は、ツール内部でデバイスコードフローによる新規認証が自動的に開始されます
4. 取得したアクセストークンと有効期限を呼び出し元に返却してください

## ペルソナ一覧

以下の3名が利用可能です:
- **大川貴志**（おおかわ たかし）: 開発担当者（実装リーダー）
- **ワタナベタナカ**: システム課長（技術側の意思決定者）
- **サトウスズキ**: 営業課長（ビジネス側のオブザーバー）

## 使用するツール

- `get_auth_token`: ペルソナ名を指定してアクセストークンを取得

## 注意事項

- 認証にはデバイスコードフロー（`--use-device-token` 相当）が使用されます
- 初回認証時はユーザーにデバイスコードとURLが表示されます
- 認証完了まで待機してからトークンを返却してください
- トークンは自動的にキャッシュされ、次回以降はサイレントに取得されます
